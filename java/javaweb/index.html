线程

实现线程的方法： 两种实现线程的方法----1、继承Thread类与实现Runnable接口，2、用 synchronized 关键字修饰同步方法
两种实现同步的方法----1、synchronized，2、wait和notify
第一种：
new Thread(){}.start();
这表示调用 Thread 子类对象的 run 方法，new Thread(){}表示一个Thread 的匿名子类的实例对象，子类加上 run 方法后的代码如下：
new Thread(){
public void run(){}
}.start();
第二种：
new Thread(new Runnable(){}).start();
这表示调用 Thread 对象接受的 Runnable 对象的 run 方法，new Runnable(){}表示一个 Runnable 的匿名子类的实例对象,runnable 的子类加上 run 方法后的代码如下：
new Thread(new Runnable(){
public void run(){}
}).start();
反对使用stop()和 suspend()方法： stop()方法是不安全的，它会解除由线程获取的所有锁定
suspend()方法容易发生死锁
sleep()和wait()： sleep----Thread 类中定义的方法，表示线程休眠，会自动唤醒；sleep()不释放同步锁
wait----Object 中定义的方法，需要手工调用 notify()或者 notifyAll()方法；wait()释放同步缩

垃圾回收

GC：垃圾回收，使用 GC 可以进行垃圾空间释放操作。
垃圾回收器的基本原理：对于 GC 来说，当程序员创建对象时，GC 就开始监控这个对象的地址、大小以及使用情况。通常，GC 采用有向图的方式记录和管理堆(heap)中的所有对象。通过这种方式确定哪些对象是"可达的"，哪些对象是"不可达的"。当 GC 确定一些对象为"不可达"时，GC 就有责任回收这些内存空间。
垃圾回收的好处：
1、可以有效的防止内存泄露；
2、有效的使用可以使用的内存。
两种垃圾回收机制：自动回收和手工调用 System.gc()方法
调用 System.gc 就相当于调用了Runtime.getRuntime().gc()方法
但是需要注意的是---程序员可以手动执行 System.gc()，通知 GC 运行，但是 Java 语言规范并不保证 GC 一定会执行。

变量和方法

局部变量：
局部变量前不能放置任何访问修饰符 (private，public，和 protected)，final 可以用来修饰局部变量。
方法：
abstract的 methods就是让子类 implement(实现)具体细节的，所以不能以private 和 final修饰。
静态方法不能直接访问非静态方法。
实例变量（成员变量）：
final修饰的实例变量（成员变量）没有默认值，必须在 constructor (构造器)结束之前被赋予一个明确的值。
例如：final int i = 0;

//遍历map中的键
for (Integer key : map.keySet())
{
    System.out.println("Key = " + key);

{
    System.out.println("Value = " + value);
}

Map map = new HashMap();

while (entries.hasNext())
{
    Map.Entry entry = entries.next();

}

懒加载：
简述：当我们查询一个对象的时候，在默认情况下，返回的只是该对象的代理对象，当用户去使用该对象的属性时，才会向数据库再一次发出查询语句。
